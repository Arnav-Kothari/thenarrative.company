03 — Functional Color

Color That Communicates

Brand colors make Katha look like Katha. Functional colors make information readable. These two systems are separate. Brand colors (amber, cream, ink) handle identity and structure. Functional colors handle meaning - they tell the viewer whether something is good, bad, done, blocked, or simply a different category of thing.

The rule is simple: if a card or box is colored, a viewer should be able to say why. Color that cannot be explained is decoration, and decoration in data is noise.

The Color Map

For every card, box, or data element, ask one question: what does this communicate to the viewer? Then pick the color from this map. There are no other considerations - not what quarter it falls in, not what category it belongs to by default, not what looks balanced on the page.

Color Name Meaning Use when... Example
Sage Positive. Good. Done. The outcome is achieved, the metric is above target, the deal is closed, the thing went well. Past, present, or future - if the card represents a positive outcome, it is sage. "Pilot completed" / "All clients at full rate" / "Revenue above target"
Clay Negative. Bad. Risk. Something is blocked, off-track, below target, or threatens progress. A problem that needs attention. A gap that could hurt. "Pipeline stalled" / "Metric below target" / "Key dependency blocked"
Amber Critical. The unlock. The single highest-leverage move. The thing that, if it happens, makes everything else possible. Use sparingly - amber is also the brand accent, so in functional contexts it should only appear on 1-2 cards per page max. If too many things are amber, nothing is critical. "Close client #2" / "Secure Series A" / "Ship v1"
Steel Neutral. In progress. FYI. No judgment yet - work is underway, context worth noting, a data point the viewer should see. Not good, not bad, just happening. "Building case study deck" / "Client #2 ramping" / "Q3 data pending"
Plum Category: people / team. Distinguishes a type of work (hiring, team-building) from other types. No judgment - just identity. For other categories beyond people, assign a color and document it per project. "Hire ops lead" / "Employee #2 - analytics"
Default No special meaning. The card exists but carries no judgment, no urgency, no category distinction. Supporting work. Operational follow-through. Default is not a safe fallback. It is a deliberate statement: "this has no special meaning." If the card IS meaningful, it needs a color. Only used when nothing above applies
Note on amber

Amber is Katha's primary brand color. It already appears on labels, CTAs, eyebrows, and section markers across every page. When amber is also used as a functional color on cards, it must clearly mean "this is the critical unlock" - not just "this is important" or "this needs emphasis." If a page already has amber in its structural elements (headers, labels), limit functional amber to 1-2 cards maximum. Otherwise the signal drowns in the noise of the brand palette.

How to Assign Color

Start with Step 0. If the entity already has a color, stop - you're done. Only use Steps 1-3 to decide what that color should be the first time an entity appears.

Step Question If yes
0 Does this entity already have a color on this page?
Same name = same color. Always. If "Salesforce" is sage on one card, every Salesforce card is sage. No exceptions.		 Use the color already assigned. Do not re-evaluate. The color was chosen based on what the entity represents overall, not what any individual card says. A Salesforce card is sage whether it says "pilot completed" or "scope expansion" - because Salesforce is the anchor client and that's a positive thing.

Only proceed to Steps 1-3 if this is the first time the entity appears and you need to decide its color.
1 Does this entity carry a judgment overall?
Looking at everything this entity represents across the whole page - is it positive, negative, or the critical unlock?		 Good outcome = sage
Bad outcome / risk = clay
The one critical unlock = amber (max 1-2 per page)
2 Does this card represent a distinct category?
Is it a different type of work that needs to be visually separated from its neighbors?		 People / hiring = plum
Other categories = assign a color, document it, use it consistently
3 Does this card need visibility without judgment?
Is it in-progress, informational, or worth noting but not good or bad?		 Yes = steel
No = default (warm cream, no color)

Rules

Rule Explanation
Color reflects meaning, not time. A roadmap card representing a positive outcome is sage whether it happened in January or is planned for December. A risk is clay whether it's active today or anticipated next quarter. Never assign default gray to a future card just because it hasn't happened yet - ask what the card communicates, not when it occurs.
Ask for every card, not just the obvious ones. The mistake is not assigning the wrong color - it's skipping the question. Walk through every card on the page and ask "what does this communicate?" Default gray is not a safe fallback. It is a deliberate statement that the card has no special meaning. If the card IS meaningful, it needs a color.
Sage and clay are reserved for judgment. Never use green or red to distinguish arbitrary categories. They are too loaded with positive/negative meaning. A "Sales" card should not be green just because you picked green for sales - that implies sales is going well.
Amber is rare. Max 1-2 per page. Amber is the brand accent - it already appears on labels, buttons, and section markers. When it also appears on a functional card, it must mean "this is the single critical unlock." If three things are amber, nothing is critical. Reserve it ruthlessly.
Same name, same color. No exceptions. If "Salesforce" is sage on one card, every Salesforce card on the page is sage. The color is assigned to the entity, not the individual card. Pick the color based on what the entity represents overall - its role in the story - and lock it in. This overrides any per-card judgment.
Max 4-5 colors per page. More than 5 colors becomes visual noise. If you have more categories than colors, some should be default. Group smaller categories into "Other."
Never rely on color alone. Always pair color with a text label, icon, or shape. The label tag on Katha cards ("DONE," "RISK," "HIRE") handles this.
Status rolls up to worst case. When summarizing multiple items, the summary takes the most severe color. If sub-items are sage, default, and clay - the summary is clay.

07 — Voice & Tone

How Katha Speaks

Katha is
  • Precise and specific — never vague or generic
  • Confident without being arrogant
  • Intellectually curious, not condescending
  • Direct — sentences are short, verbs are strong
  • Platform-native — understands where the audience lives
  • Warm, human, and earned — not polished or over-managed
Katha is not
  • Corporate jargon or buzzword-heavy
  • Overly formal or stiff
  • Vague about what it actually does
  • Hyperbolic or chest-thumping
  • Passive — never hedges unnecessarily
  • Long-winded — makes the point and moves on
